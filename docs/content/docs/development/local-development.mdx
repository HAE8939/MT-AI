# 本地开发

主应用是 `web/` 纯静态前端，AI 请求由浏览器直连用户自己配置的接口，没有项目后端。开发行为约定见仓库根目录 `AGENTS.md`。

## 1. 启动前端

```bash
cd web
bun install
bun run dev
```

前端默认访问：

```text
http://localhost:3000
```

常用命令：`bun run typecheck` 类型检查，`bun test <file>` 跑单测（如 `bun test src/lib/canvas/local-workflow.test.ts`）。

## 2. 配置模型

打开右上角配置弹窗，填写自己的 `Base URL`、`API Key` 和模型名。提示词、素材、COS 上传都由浏览器前端直连。

## 3. 本地 Canvas Agent（可选）

需要调试 Codex / Claude Code 操作画布的链路时：

```bash
cd canvas-agent
npm install
npx tsc -p tsconfig.json
node dist/index.js
```

改动 `canvas-agent` 源码后需重新构建；已注册 MCP 的话还要重启 Codex。Windows 下 `start-all.bat` 可同时拉起 Agent 与前端。

## 4. 启动文档站

文档站是 `docs/` 下独立的 Next.js（Fumadocs）应用：

```bash
cd docs
bun install
bun run dev
```

## 常见场景

- 改画布、页面和交互：`web/src/pages/`、`web/src/components/canvas/`
- 改外部服务请求（生图 / 视频 / 音频 / COS / 画廊）：`web/src/services/api/`
- 改全局状态（渠道配置、提示词、素材、任务）：`web/src/stores/`
- 改本地 Agent 与 MCP 工具：`canvas-agent/src/`
- 改文档站内容：`docs/content/docs/`
