# 配置指南

配置入口有两个：右上角配置弹窗和 `/config` 独立页面，两者复用同一套面板，共六个分区：**渠道 / 模型 / 偏好 / 腾讯云 COS / RunningHub / Codex**。所有配置保存在浏览器本地。

## 渠道

支持添加多个模型渠道，每个渠道包含 Base URL、API Key 和调用格式：

- **调用格式**：OpenAI 兼容（默认）或 Gemini，切换默认格式时同步更新默认 Base URL。Gemini 格式支持拉取模型列表、文本问答、生图和图生图；音频与视频生成仍需 OpenAI 兼容渠道。
- **东木-AI 聚合平台**：一键添加，模型列表与能力分类（文本 / 图片 / 视频 / 音频）自动发现，生图与视频走平台统一异步任务接口。
- **火山方舟 Seedance**：按兼容渠道方式配置，Base URL 填 `https://ark.cn-beijing.volces.com/api/plan/v3`。该接口不提供 `/models` 列表，需手动填写模型名（如 `doubao-seedance-2.0`）。
- Base URL 已以 `/v1`、`/api/v3` 或 `/api/plan/v3` 结尾时不会再追加 `/v1`，反代地址可直接使用。

使用 **New API** 时，在其 `系统设置 → 聊天方式` 中添加 `https://你的部署地址?apiKey={key}&baseUrl={address}`，跳转后自动写入渠道配置。

## 模型

对每个渠道「拉取模型列表」后，在此为四类能力分别设置默认模型：文本、图片、视频、音频。模型值按 `渠道::模型名` 组织，画布节点可在生成面板中单独覆盖。

## 偏好

生成偏好的全局默认值：图片质量、比例、数量、画布批量张数；视频比例、清晰度、时长、生成声音、水印；音频音色、格式、语速等。画布生成配置节点里的设置优先于全局默认。

## 腾讯云 COS

媒体文件的可选云同步，采用**本地优先上传队列**：

- 配置项：SecretId、SecretKey、Bucket、Region、公网访问域名、对象前缀，提供连接测试。
- 图片、视频、生成结果与素材先落浏览器本地，再进入上传队列；失败自动重试三次，顶部同步中心可查看状态并手动重试。
- 局域网 HTTP 访问（非安全上下文）时签名自动降级为纯 JS 实现，功能不受影响。
- 安全建议：使用 CAM 子账号密钥，只授予单桶单前缀的 PutObject / GetObject / DeleteObject 最小权限；公有读桶的直链等于公开内容。

## RunningHub

云工作流平台连接：Base URL（默认国际站 `https://www.runninghub.ai`）与 API Key。用法见[工作流使用指南](/docs/canvas/workflows-guide)。

## Codex

本地 Canvas Agent 连接管理：

- 展示插件安装与本地 Agent 启动步骤（见 [Codex App 插件](/docs/overview/codex-app-plugin)）。
- 可配置本地 Agent 地址（默认 `http://127.0.0.1:17371`）、Connect token 和「工具执行确认」开关。
- 连接状态同步到画布与顶部状态图标，刷新页面后按本地保存的地址和 token 自动重连。

## 管理员配置下发（config.json）

团队部署时，编辑部署产物中的 `config.json`（源码位于 `web/public/config.json`）可向所有打开站点的设备下发配置：

- **渠道**（`channels`）：预置渠道列表与默认模型；管理员渠道不会混入用户自建渠道的本地持久化。
- **腾讯云 COS**（`cosConfig`）与 **RunningHub**（`runninghub`）：字段与配置页一致，可部分覆盖。
- 其余生成偏好字段（质量、比例、数量、视频与音频参数等）同样可下发为默认值。

站点启动时拉取该文件并按内容哈希检测变更，变更后自动应用。注意：`config.json` 是公开可下载的静态文件，写入其中的 Key 对所有访问者可见，只适合内网或可信团队环境。
