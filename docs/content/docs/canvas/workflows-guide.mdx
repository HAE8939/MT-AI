# 工作流使用指南

工作流模块把「画布上跑通的生成过程」沉淀为可复用的资产。共有三种来源，统一在 `/workflows` 页管理，在画布侧栏「工作流」tab 或运行弹窗中执行，结果都写回画布节点并进入任务中心。

| 来源 | 执行位置 | 依赖 |
| --- | --- | --- |
| RunningHub 云工作流 | 云端（ComfyUI 级管线） | RunningHub 账号与 API Key |
| 画布模板 | 画布本地（走已配置渠道） | 无 |
| 本地工作流 | 画布本地（多步自动串跑） | 无 |

## RunningHub 云工作流

### 前置条件

- 在 [RunningHub](https://www.runninghub.ai) 上把目标工作流**手动成功运行过一次**，否则 API 无法调用。
- 在「配置 → RunningHub」填入 Base URL（默认国际站 `https://www.runninghub.ai`）和 API Key；登记弹窗里也可填写，两处同源。

接口走 OpenAPI v2（`Authorization: Bearer` 鉴权）：提交 `POST /openapi/v2/run/workflow/{workflowId}`、查询 `POST /openapi/v2/query`、上传 `POST /openapi/v2/media/upload/binary`。

### 登记工作流

1. 在 `/workflows` 页点击登记，填入 workflowId。
2. 配置参数映射：把工作流中需要暴露给用户的 nodeId / fieldName 登记为输入参数，类型支持**图片、提示词、数值**。
3. 推荐用「导入 JSON」：粘贴 RunningHub 的「导出工作流 API」JSON，自动识别参数（`LoadImage` → 图片、`CLIPTextEncode` → 提示词并带出默认值）。

内置模板「Z Image 亿级像素文生图」开箱即用：参数留空即按工作流默认提示词出图，可用来验证链路。

### 运行

画布侧栏「工作流」tab 点击云工作流卡片，直接进入运行视图：

- 图片参数双来源：选画布上任意图片节点（带缩略图下拉，选中节点自动预填第一个图片字段），或本地上传（≤30MB，提交时经上传接口换取文件名）；公网 URL 直接透传。
- 数值参数渲染为步进器，带默认值提示。
- 「立即运行」后停留在运行视图可迭代重跑；状态条实时跟踪，失败原因透传展示。
- 结果自动写回画布占位节点，同时可在任务中心查看「云工作流」任务。

## 画布模板

把一组画布节点保存为**参数骨架**，之后在任意画布实例化换输入重跑：

1. 画布多选节点 → 工具栏「保存」→ 选「画布模板」。
2. 保存时按连线拓扑排序：生成节点转为步骤，无上游连线的图片 / 文本节点声明为模板输入，提示词进入步骤参数（运行前可编辑）。
3. 在 `/workflows` 页点「在画布中运行」：模板在当前画布重建节点与连线，填充输入槽后逐步执行，结果写回对应节点。

## 本地工作流

把画布上跑通的**多步本地生成**（文生图 → 参考图编辑 → 放大…）保存为可一键串跑的流程，不依赖云端：

1. 画布上搭好并跑通多步生成链（例如：文本 → 生成配置 1 → 图片 → 生成配置 2 → 图片）。
2. 多选这些节点 → 工具栏「保存」→ 选「本地工作流」，系统自动识别输入槽（无上游产出的空图片节点）。
3. 画布侧栏「工作流」tab 进入运行面板，为每个输入槽填图 → 「立即运行」。
4. 步骤按依赖顺序自动串跑：每步产出自动作为下一步参考图，进度显示「第 N/M 步」，最终结果写回占位节点。
5. 某一步失败时停止后续步骤，并提示「第 N 步生成失败」。

当前 v1 支持图片链（各步产物为图片）；步骤执行走当前配置的生成渠道。

## 与文档智能体的关系

原「专业角色」已迁移为**文档智能体**：读取选中节点内容，用角色 System Prompt 产出连接的分析文本节点。入口在画布工具栏，管理在角色（文档智能体）列表中，与本页的生成类工作流互不影响。
