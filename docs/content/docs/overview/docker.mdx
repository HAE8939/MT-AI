# Docker 部署

如果你希望在自己的机器或服务器上运行项目，可以基于仓库根目录的 `Dockerfile` 构建镜像（nginx 静态托管前端构建产物）。

## 本地构建镜像

```bash
git clone https://github.com/HAE8939/MT-AI.git
cd MT-AI
docker build -t mt-ai .
docker run -d --name mt-ai -p 3000:3000 mt-ai
```

启动后访问：

```text
http://localhost:3000
```

Render 部署（`render.yaml`，`runtime: docker`）使用的也是这份 Dockerfile。

## 文档站镜像

文档站位于 `docs/` 目录，按带服务端能力的 Next.js standalone 应用单独构建，不打进主应用镜像。

基于当前源码本地构建：

```bash
cd docs
docker compose -f docker-compose.local.yml up -d --build
```

## 数据说明

主应用镜像由 nginx 托管静态构建产物。画布、我的素材、生成记录和 AI API Key 默认保存在浏览器本地，不需要额外挂载数据目录。
