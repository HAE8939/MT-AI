# 工作流临时蒙版编辑设计

## 目标

将提示词引擎工作流中的独立蒙版图片上传改为基于原图的临时绘制。该行为适用于所有声明了蒙版输入的工作流。蒙版始终可选：用户未绘制蒙版时允许直接运行，由提示词描述需要修改的部位，并按无蒙版的整图编辑处理。

## 交互

运行面板继续先展示原图输入，支持本地上传和选择画布图片。原图下方展示独立的“蒙版（可选）”卡片：

- 没有原图时卡片禁用，提示“请先上传原图”。
- 已有原图且没有蒙版时，点击卡片打开大尺寸蒙版编辑器；用户也可跳过蒙版直接运行。
- 保存蒙版后，卡片展示已绘制状态，并提供“重新绘制”和“清除”。
- 更换或清除原图时自动清除旧蒙版，避免图片内容和尺寸错配。

蒙版编辑器以项目现有局部重绘编辑器为基础。工作流模式只保留画笔、橡皮擦、笔刷大小、撤销、重做、清除、取消和确认，不展示修改要求、参考图、对比小图或直接生成按钮。确认只返回 PNG 蒙版，不创建素材或画布节点。

## 数据流

`usePromptEngineRun` 不再维护蒙版上传文件，而是维护编辑器生成的临时 `maskDataUrl`。本地原图使用对象预览地址打开编辑器；画布原图使用节点内容地址打开编辑器。提交时画布原图仍按现有逻辑解析为 data URL。

运行时：

1. 解析并校验原图、参考图和临时蒙版大小。
2. 有蒙版时，将蒙版传给 `runPromptEngineWorkflow`，最终由 `requestEdit` 发送局部编辑请求。
3. 无蒙版时不传 `mask`，仍把原图作为参考图调用编辑接口，实现整图编辑。

## 配置规则

所有内置提示词引擎工作流中的 `inputSpec.mask: "required"` 改为 `"optional"`。运行校验不再因缺少蒙版而阻止提交。类型、配置和界面统一表达“只要出现蒙版输入，它就是可选项”。

## 组件边界

- 共享蒙版编辑器负责在原图上绘制、维护撤销/重做历史并输出蒙版 PNG。
- 画布局部重绘入口继续负责提示词、参考图、羽化、矩形选择和生成行为。
- 工作流运行面板负责原图与蒙版状态、打开编辑器、清除蒙版和提交输入。
- 工作流执行器只负责依据是否存在 `mask` 决定是否携带蒙版，不处理 UI 状态。

## 异常处理

- 原图读取失败时不打开编辑器，并沿用现有图片读取错误提示。
- 没有有效涂抹区域时“确认”不可完成，并提示先涂抹区域。
- 原图变化立即丢弃蒙版，不尝试缩放或迁移旧蒙版。
- 蒙版或图片超过现有 10MB 限制时阻止运行并显示对应错误。
- 关闭编辑器不保存本次改动，保留此前已确认的蒙版。

## 测试

- 单元测试：所有蒙版工作流允许缺少蒙版；有蒙版时执行器仍将其传给编辑接口。
- 状态测试：选择原图后可打开编辑器；更换或清除原图会清除蒙版；确认编辑器会保存临时蒙版。
- 编辑器测试：空蒙版不能确认；绘制后可确认；撤销和重做正确恢复涂抹状态。
- 浏览器验证：分别使用本地原图和画布原图完成蒙版绘制；有蒙版与无蒙版两种运行路径均可提交。
